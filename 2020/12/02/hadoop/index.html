先拍个开始快照（关机快照只会保存硬盘状态，开机快照还会保存内存状态，故一定要关机快照）

安装hadoop、java，搭建伪分布式

xshell远程连接，关闭防火墙，service sshd start，sudo yum install openssh-server；

使用xftp传输jdk和hadoop安装包，到/opt/software

先删除之前的java包，rpm -qa | grep java，rpm -qa |grep java |xargs sudo rpm -e –nodeps；

解压，tar -zxvf jdk-8u144-linux-x64.tar.gz -C /opt/module/；z gz ，x解压，v信息，f指定文件。 -C 输出目录

hadoop解压同理；ll？？？

配置环境变量，

sudo vim /etc/profile

#JAVA_HOME
export JAVA_HOME=/opt/module/jdk1.8.0_14
export PATH=$PATH:$JAVA_HOME/bin

#HADOOP_HOME
export HADOOP_HOME=/opt/module/hadoop-2.7.2
export PATH=$PATH:$HADOOP_HOME/bin:$HADOOP_HOME/sbin

source /etc/profile

java -version

如何远程拷贝？一台一台安装hadoop java太麻烦了，

cp -r source destination

#远程吧hadoop101上的/opt/module拷贝到其余主机相同位置；

scp -r hadoop101:/opt/module/hadoop-2.7.2 (source) hadoop102:/opt/module(destination)

//但是权限和时间等没有拷贝，不是完成拷贝；

//完全拷贝是rsync；只能一个远程一个本地地址；这个更好

rsync -av hadoop101:/opt/module/hadoop-2.7.2 /opt/module

集群分发脚本，xsync；看教程文档；

配置hdfs

hdfs简介，优缺点，架构等，通过多副本机制实现高容错性；处理文件并不是以文件为的单位，而是以块为单位；

hdfs的shell操作；

上传：hdfs dfs -put etc /；hadoop fs -put etc /都行；等等命令很多；前缀加基本命令；

hdfs-hdfs

  • hadoop fs -cp /etc /etc2；
  • hadoop fs -rm -r /etc2;
  • hadoop fs -df -h;
  • hadoop fs -du -h /;disk usage;

本地->hdfs

  • put
  • copyFromLocal与上同
  • moveFromLocal
  • appendToFile;hadoop fs -appendToFile 本地文件 hdfs中文件路径

hdfs->本地

  • get;hadoop fs -get /output ./ ;/output 下载到当前目录
  • getmerge，hadoop fs -getmerge /text/* merge.txt;
  • copyToLocal，与get同；

hadoop fs -tail -f hdfs中文件；//还有其他一些常规命令；
